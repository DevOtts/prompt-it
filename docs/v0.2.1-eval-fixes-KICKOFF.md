## How to run + test (the load-bearing harness — this is what the note exists to carry)
- **Run one sample:** `EVAL_MODEL=<tier> ./eval/scripts/run-eval.sh S13 eval/runs/<gen>` — spawns a **fresh headless `claude -p` session** that loads the installed plugin and invokes `/prompt-it` on the sample text. Output → `eval/runs/<gen>/S13.md`.
- **Judge:** read the output against `eval/expected-prompts.md` (that sample's checklist + G1–G6). For batches, dispatch sonnet judge agents (5 samples each) writing evidence-quoted PASS/FAIL + per-failure diagnosis (SKILL-DEFECT / ORACLE-DEFECT / HARNESS-ARTIFACT).
- **CRITICAL GOTCHA #1 — never eval via Agent-tool subagents.** Subagents inherit the parent session's skill roster, snapshotted *before* the plugin was (re)installed → the plugin won't trigger → 20/20 false TRIGGER-FAIL. Only fresh top-level `claude -p` sessions (i.e. `run-eval.sh`) load the installed plugin. This cost a full wasted 20-agent batch.
- **CRITICAL GOTCHA #2 — the deploy loop after EVERY SKILL.md fix** (a plain reinstall silently reuses a STALE cache — the fix won't be live):
  ```bash
  # 1. edit plugins/prompt-it/skills/prompt-it/SKILL.md, then mirror to root:
  cp plugins/prompt-it/skills/prompt-it/SKILL.md SKILL.md
  # 2. commit + push
  git add -A && git commit -m "fix(skill): ..." && git push origin main
  # 3. PURGE the cache (this is the step people miss):
  rm -rf ~/.claude*/plugins/cache/devotts/prompt-it
  # 4. refresh + reinstall
  claude plugin marketplace update devotts && claude plugin install prompt-it@devotts
  # 5. CONFIRM the fix is in the installed cache (case-insensitive!):
  grep -ric "your new rule text" ~/.claude*/plugins/cache/devotts/prompt-it/*/plugins/prompt-it/skills/prompt-it/SKILL.md
  # 6. only now rerun the samples
  ```
- **CRITICAL GOTCHA #3 — verification greps must be `grep -i`.** A case-sensitive grep ("first character" vs the file's "FIRST character") nearly triggered a needless cache-purge loop. Copy exact text or use `-i`.
