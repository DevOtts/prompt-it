## Cliente / Clientes — term resolution rule

"Cliente" refers to two distinct entities in this tenant's data. Before answering
any question containing "cliente"/"clientes", resolve which one applies from the
question's own wording — never default to one sense by habit.

1. **Org record (type=client)** — an internal account/organization entity.
   Applies to questions about accounts, contracts, invoicing, or account
   management (e.g. "quantos clientes ativos temos?", "quais clientes
   renovaram o contrato?").
2. **Shopify buyer** — a person who purchased from the tenant's Shopify store.
   "Cliente recorrente" / "cliente repetido" = a buyer with 2+ orders in the
   queried period. Applies to questions about purchases, orders, recorrência,
   vendas, or revenue (e.g. "quantos clientes recorrentes tivemos este mês?",
   "quantos clientes compraram mais de uma vez?").

Resolution rule: if the question contains purchase/order/revenue vocabulary
(compra, pedido, venda, recorrente, Shopify), resolve to sense 2 (Shopify
buyer) even though the literal word is "cliente." Only resolve to sense 1
(org record) when the question is about the account/contract relationship
itself.

If this tenant has other data entities also labeled "cliente" beyond these
two, apply the same rule: infer from the domain vocabulary in the question,
not from string-matching "cliente" to one hardcoded entity.

### Worked examples

| Question | Resolution | Expected behavior |
|---|---|---|
| "Quantos clientes recorrentes tivemos este mês?" | Shopify buyer, recorrente = 2+ orders this month | Query Shopify orders grouped by buyer for the current month; count buyers with ≥2 orders. Never query org type=client for this question. |
| "Quantos clientes ativos temos?" | Org record, type=client | Query orgs where type=client and status=active. |
| "Quantos clientes recorrentes tivemos este mês?" (zero repeat buyers exist) | Shopify buyer | Answer "Nenhum cliente recorrente (comprador com 2+ pedidos) encontrado este mês." — never the bare "no clients found" against the wrong table. |

### Missing data / ambiguity behavior
Never answer a bare "no clients found" / "nenhum cliente encontrado" without
naming which interpretation produced the zero result. If the question could
plausibly resolve either way, state the interpretation used and offer the
other reading, e.g.: "Interpretei 'clientes' como compradores recorrentes do
Shopify (2+ pedidos). Nenhum registro encontrado neste mês. Quer que eu
busque contas do tipo cliente (CRM) em vez disso?"

### Response format
Always state which "cliente" sense was used before giving the number or
list. One short clause, not a preamble paragraph.

### Out of scope
Do not change how any other tenant entity (leads, campaigns, contracts,
invoices) is resolved. Do not alter the underlying Shopify sync or CRM
query logic — this is a term-resolution instruction only.
